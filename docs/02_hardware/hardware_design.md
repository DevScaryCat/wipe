## 2. 핵심 메커니즘: 슬라이드 트레이 + 수직 리프트

> ⚠️ **폰을 기계가 집어삼키는 방식(롤러/세로 크레들 강제 고정)은 파손 리스크로 전면 폐기.**
> 사용자가 트레이에 폰을 **눕혀서 올려놓는** 방식만 채택한다. (RFP 3.2 조건)

### 동작 시퀀스

```
[1단계] 트레이 인출 & 폰 안착
  결제 완료 → 전면 하단 도어 열림 + 평평한 트레이가 수평으로 슬라이드 인출
  → 사용자가 폰을 눕혀서 올려놓음 (케이스/그립톡 상태 그대로)
  → 트레이가 기기 내부로 닫혀 들어감

[2단계] 이물질 2중 필터링 (트레이 진입 직후)
  → Load Cell: 무게 150~350g 범위 벗어나면 트레이 재배출
  → AI 비전(RPi5 + USB캠): "스마트폰 형태인가?" 0.5초 판정, 아니면 재배출

[3단계] 리프트 상승
  NEMA17 + T8 리드스크류가 트레이를 통째로 투명 챔버 정중앙까지 수직 이동
  → 리미트 스위치로 상단 도달 + 챔버 밀폐 확인

[4단계] 살균 시작 (밀폐 확인 후에만)
  UV-C LED 점등 → 사용자가 투명 패널 너머 파란빛 속 폰 직접 목격
  → 하단 스크린: 세균 카운터 애니메이션
  → 30초 타이머 진행

[5단계] 완료
  UV-C LED OFF → 리프트 하강 → 도어 열림 + 트레이 배출
  → 폰 회수 → 완료 화면 + SNS 공유 QR
```

## 3. 수직 리프트 상세 설계

### 방식: T8 리드스크류 (트레이 플랫폼 승강)

```
[NEMA17 스테핑 모터]
       │
[T8 리드스크류 300mm]   ← 회전 → 직선 운동 변환
       │
[리드스크류 너트] ──── [트레이 플랫폼]
                              │
                       [슬라이드 트레이]
                       (폰을 눕혀 올리는 평판 + Load Cell)

가이드: 8mm 선형 레일 × 2 (좌우 각 1개)
베어링: LM8UU × 4
```

**선택 이유:**
- 정밀 위치 제어 가능
- 셀프 록킹 (역회전 방지 → 트레이가 폰 무게에 안 내려옴)
- Arduino + A4988으로 제어 가능

### 슬라이드 트레이

```
   ┌─────────────────────┐
   │   📱 (폰을 눕힘)     │   ← 평평한 받침, 케이스/두께 무관
   └─────────────────────┘
   [Load Cell 무게 센서] ← 트레이 하단
   진입부 상단에 USB 비전 카메라 마운트

폰 대응 범위: 전 기종 (눕힘 방식이라 폭/두께 제약 거의 없음)
재질: 3D 프린팅(PLA/PETG) → 양산 시 사출
수평 인출: 별도 소형 스테핑/기어드 모터 또는 솔레노이드 도어
```

## 4. UV-C 챔버 설계

```
후면 (흰색 반사판 + UV 반사 알루미늄 테이프)
┌─────────────────────────────┐
│  💡UV-C  💡UV-C  💡UV-C    │  ← UV-C LED × 3 (상단)
│                             │
│      📱 폰 (트레이 위)      │  ← 상·하면 동시 조사
│                             │
│  💡UV-C  💡UV-C  💡UV-C    │  ← UV-C LED × 3 (하단)
└─────────────────────────────┘
전면 (투명 아크릴/PC 5mm)

UV-C LED 스펙: 265~270nm, 4W, 방사각 120°
챔버 내부: 흰색 반사판 + 알루미늄 테이프 → 효율 극대화
```

> ⚠️ 안전 인터락: 리프트 상단 도달 + 챔버 밀폐(리미트 스위치 확인) 후에만 UV-C 점등.
> UV-C는 눈/피부에 치명적. 챔버 밀폐 = 절대 조건.
>
> 🔬 **검토 필요:** 30초 살균으로 목표 살균력(log reduction)을 확보하려면 LED 출력·개수·조사거리 실측 검증이 필요하다. (PROGRESS "초기 검토 과제" 참조)

## 7. 안전 설계 원칙

| 위험 요소 | 대책 |
|---|---|
| UV-C 노출 | 리미트 스위치 인터락 — 챔버 완전 밀폐 확인 후만 점등 |
| 폰 낙하/손상 | 트레이에 눕혀 안착(강제 클램핑 없음) + 평판 지지 |
| 이물질 투입 | Load Cell 무게 + AI 비전 2중 필터링 → 비정상 시 트레이 재배출 |
| 과전류 | 퓨즈 + 모터 드라이버 전류 제한 |
| 비상 정지 | 비상 스위치 → Arduino 인터럽트로 즉시 UV-C 차단 |

## 8. (Phase 2 대비) 살균수 모듈 선반영

> Phase 1은 **UV-C 살균 전용**으로, 액체 세척 기능은 **작동하지 않는다.**
> 단, 향후 Phase 2(세척 모듈) 추가를 위해 하단부에 3% 과산화수소수 탱크 공간과
> 배관 경로를 기구설계 단계에서 미리 확보한다. (`cleaning_fluid_spec.md` 조건 준수)
> - 벤팅 캡(가스 배출), 불투명 탱크(빛 차단), 유로 내 Fe/Cu/황동 접촉 금지
