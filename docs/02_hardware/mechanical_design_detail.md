| 항목 | 결정 |

## 1. 전체 레이아웃 (Z축)

```
 1500 ┬───────────────┐
      │  [상단] 투명    │  ← 투명 UV-C 챔버 (PC 3면 + UV LED)
      │   UV-C 챔버    │     폰이 여기로 올라와 살균 · 발광 비콘
 1100 ┼───────────────┤
      │  [중앙] 10"     │  ← 세로 터치스크린 (세균 UI) + 결제·QR
      │   스크린+결제   │     눈~가슴 높이
  700 ┼─ 트레이 투입구 ─┤  ← 슬라이드 트레이 (허리 높이)
      │  [하단] 제어부   │  ← RPi5 · Arduino · SMPS (+Phase2 탱크)
   80 ┼───────────────┤
    0 ┴── 캐스터 ──────┘
```

| 영역 | Z 범위(mm) | 내용 |
|---|---|---|
| 상단 — 챔버 | 1100 ~ 1500 | 투명 챔버, UV-C 살균, 폰 도착 위치(비콘) |
| 중앙 — UI | 740 ~ 1080 | 10" 세로 터치 + 결제 단말 + QR |
| 투입구 | ~700 | 슬라이드 트레이 인출구(허리) |
| 하단 — 제어 | 80 ~ 680 | RPi5·Arduino·SMPS, (Phase2) 과산화수소수 탱크 |
| 캐스터 | 0 ~ 80 | 이동 바퀴 ×4 |

## 2. 파트 상세

### 2.1 외형 / 외함 (★미학 핵심)
> "스티브 잡스 공기방울 테스트" = 내부에 낭비 공간 0, 외부는 군더더기 0.

- **소재/마감:** 무광 화이트(Matte White) 바디. 오염에 강하고 고급감.
- **유니바디:** 외부 **나사선·파트 분할선 안 보이게** (Snap-fit/내부 체결/숨김 나사). 모서리 라운드.
- **전면은 딱 3요소:**
  1. **챔버 아퍼처** — 상단의 빛나는 투명 창 (히어로)
  2. **플러시 블랙 스크린** — 면과 단차 없이 매립
  3. **미니 슬롯** — 트레이 투입구 (얇은 라인)
  그 외 버튼·라벨·이음새 최소화.
- **비율:** 폭·깊이를 **최소 슬림**하게(토템). 내부 부품을 밀집시켜 빈 공간 제거.
- **제작:** 프로토 = 3D프린팅/레이저컷 패널 또는 기성 토템 케이스 리트로핏 / 양산 = 판금or사출 **외주**.

### 2.2 투명 UV-C 챔버 (상단)
- **구조:** 전/좌/우 3면 **투명 PC(폴리카보네이트) 5mm**, 후면 흰색 반사판 + UV 반사 알루미늄 테이프.
- **광원:** UV-C LED **265~270nm, 4W ×6** (상·하 배열 → 폰 상·하면 동시 조사).
- **밀폐/안전 (절대조건):** 챔버 입구를 트레이가 막아 밀폐 → **상단 리미트 스위치 ON 확인 후에만 UV 점등**. UV-C는 눈·피부 유해 → 누출 0 설계.
- **열린 과제:** 폰을 트레이에 눕히면 **바닥면(트레이 접촉면)은 그림자** → 양면 살균 위해 (a)돌기로 띄움 (b)윗면 위주 인정 (c)석영 트레이 중 택1. (살균력 실증과 함께)

### 2.3 슬라이드 트레이 + 수직 리프트 (★핵심 메커니즘)
**(A) 트레이 슬라이드(수평):** 도어 열림 → 평판 트레이 인출 → 폰 **눕혀 안착**(케이스 그대로) → 닫힘. 트레이 내부 ~**180×90** (전 기종). 롤러로 집어삼키는 방식 금지.

**(B) 리프트(수직):** 트레이 캐리지가 투입구(~700) → 챔버(~1300)까지 **약 600mm 상승**.
- **구동:** **GT2 벨트 + 풀리**(장행정 권장) 또는 T8 리드스크류 600mm + 안티백래시 너트.
- **모터:** NEMA17 (17HS19) + 드라이버(A4988/TMC2209).
- **가이드:** 8mm 환봉 ×2 + LM8UU ×4 (또는 MGN12 리니어레일 ×2) — **듀얼레일로 강성·정렬 확보**.
- **캐리지:** 트레이 플랫폼 + **Load Cell + HX711**(무게 1차 이물 필터).
- **리미트:** KW12-3 ×2 — 하단(홈/원점) + 상단(=UV 인터락).
- **케이블:** 움직이는 캐리지 → **케이블 체인**으로 관리.

### 2.4 스크린 + 결제 (중앙)
- **디스플레이:** 10인치 **세로형** 터치 800×1280, HDMI + USB(터치).
- **결제:** **VAN 직가맹 카드단말(CAT/리더기)** + QR 스캔. (PG 웹결제 회피 — `../01_business/payment_fee_analysis.md`)
- **마운트:** 전면 패널에 매립(플러시), 30° 미만 평면.

### 2.5 제어부 / 전원 (하단)
- **연산:** Raspberry Pi 5 (UI·AI비전·통신).
- **실시간 제어:** Arduino Mega (모터·UV·센서·인터락). *MCU 최종 미정 — Mega 기본, ESP32 옵션.*
- **전원:** SMPS 220V → 12V(모터·UV) / 5V(보드). **퓨즈** + 드라이버 전류제한.
- **(Phase2)** 3% 과산화수소수 탱크 공간 선반영(벤팅캡·불투명·Fe/Cu 비접촉).

### 2.6 프레임 (골격)
- **알루미늄 프로파일(2020/3030)** 로 뼈대 조립 (직접 설계 X — 잘라서 볼트). 3D프린터·CNC와 동일 방식, 튼튼·빠름.
- 패널은 프레임에 부착.

## 4. 구현 난이도 / 리스크

| 리스크 | 내용 | 대응 |
|---|---|---|
| **600mm 장행정** | 리드스크류 휨·진동, 캐리지 정렬 | 벨트 구동 + 듀얼레일 + 강성 캐리지 |
| **UV-C 차폐·밀폐** | 누출 시 인체 유해, 인증 직결 | 밀폐 인터락 이중화, 차폐 구조 |
| **폰 안정 유지** | 트레이가 폰 들고 600mm 상승·30초 유지 | 평판 + 미끄럼 방지, 비강제 |
| **케이블 관리** | 움직이는 캐리지 배선 | 케이블 체인 |
| **30초 살균력** | 출력·거리로 목표 달성? + 그림자면 | 실측 검증 (열린 과제) |
| **미학 vs 메커니즘** | 긴 리프트축이 내부 차지 → "공기방울"(빈 공간/비대) | 슬림·밀집 설계, 단행정 대안 검토 |

## 5. 미학 원칙 (Jobs 테스트)

1. **내부 밀집** — 리프트축·전장 빈틈없이. "물에 넣어 공기방울 0".
2. **외부 3요소** — 챔버 / 플러시 스크린 / 미니 슬롯. 그 외 안 보이게.
3. **나사선 0 · 유니바디 · 라운드** — 이음새·체결 숨김.
4. **슬림 비율** — 폭·깊이 최소. 챔버=발광 비콘.
