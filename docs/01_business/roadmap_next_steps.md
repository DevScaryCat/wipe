## 1. 단계별 플랜

### Phase A — 데모 마무리 & 수요 검증 (지금 ~ 3주) ★최우선·저비용·본인 강점
- [ ] 데모 카피 정합("30초"→실제 흐름), **자동 순환 "발표 모드"**, 데모 영상 녹화
- [ ] (선택) 실제 폰 사진 합성 비포/애프터 목업
- [ ] **수요 검증 (가장 중요):**
  - [ ] 잠재 사용자 인터뷰 + "500원에 쓸래?" 설문 (목표 30~50명)
  - [ ] 성수/홍대 카페·팝업 **점주 3~5곳 미팅 → "무료로 한 대 둬볼래?"** (파일럿 입점 의향)
  - [ ] → **GO/NO-GO·가격·입지** 판단 (리스크 문서의 최대 가설 해소)

### Phase B — 사업 기본 세팅 (Phase A와 병행)
- [ ] 사업자등록
- [ ] **토스페이먼츠 / VAN 직가맹 문의** (수수료·정산조건 확인)
- [ ] 인증 풀스코프(KC 전기안전 + EMC 전파 + 공장심사) **견적만** 확보 → 예산 재산정

### Phase C — 하드웨어 프로토타입 (부품 도착 후, 학습 병행) ※임베디드 약점 구간
- [ ] 아두이노: 스테핑모터 + T8 리드스크류 + 리미트스위치 → **리프트 1축** 제어
- [ ] 슬라이드 트레이 + 이물질 감지(Load Cell + AI 비전) 목업
- [ ] **UV-C 챔버 + 안전 인터락**(밀폐 확인 후 점등)
- [ ] RPi5 ↔ 아두이노 UART, 미들웨어 mock → 실제
- [ ] **30초 살균력 실측 검증** (LED 출력·거리 — 열린 과제) / 폰 양면 노출 방식
- [ ] 기구 설계 외주(RFP 작성 완료) 발주

### Phase D — 통합 MVP (목표 2026.10)
- [ ] **MVP UI = 살균 전용 단일 단계로 정리** + Tauri 패키징
- [ ] UI ↔ 미들웨어 ↔ 펌웨어 통합 (localhost 이벤트)
- [ ] 결제 연동 (VAN 직가맹)
- [ ] 백엔드: Supabase **telemetry(원격 모니터링)** + 관리자 대시보드(Next.js)
- [ ] 500사이클 무중단 내구 테스트

### Phase E — 파일럿 → 인증 → 배치 (~2027.Q1)
- [ ] 지인 카페 비공개 파일럿 (인증 전), KPI 측정(이용수·전환·재방문·공유율)
- [ ] KC + EMC 인증 제출·보완
- [ ] 성수/홍대 정식 배치
